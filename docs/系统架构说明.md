# 晋彩智绘 — 系统架构与模型说明

## 一、系统架构

### 1.1 整体架构

```
┌─────────────────┐     JWT      ┌──────────────────┐   OpenAI API   ┌─────────────────┐
│   Vue 3 前端     │ ──────────▶ │  Spring Boot 后端  │ ────────────▶ │  AI Gateway     │
│   (Port 5173)   │ ◀────────── │   (Port 8080)     │ ◀──────────── │  (Port 4000)    │
└─────────────────┘    JSON      └──────────────────┘   JSON         └─────────────────┘
                                         │                                    │
                                         ▼                                    ▼
                                  ┌──────────────┐                   ┌──────────────────┐
                                  │   MySQL 8    │                   │  上游 AI 服务商    │
                                  │(jincai_zhihui)│                   │ DeepSeek/通义等   │
                                  └──────────────┘                   └──────────────────┘
```

### 1.2 三层解耦设计

1. **前端层**：Vue 3 + TypeScript + Pinia 状态管理，负责 UI 渲染和用户交互
2. **业务层**：Spring Boot 3 + Spring Security + JPA，负责认证鉴权、业务逻辑、数据持久化
3. **AI 路由层**：New API Gateway (Docker)，统一管理多家 AI 服务商的 API Key、路由策略和重试机制

后端不直接调用任何 AI 服务商 API，所有 AI 请求统一通过 Gateway 的 OpenAI 兼容接口转发。

## 三、核心模块说明

### 3.1 认证与鉴权

- 基于 JWT 的无状态认证
- 角色分为 TEACHER 和 STUDENT
- 路由级别权限控制：`/api/teacher/**` 仅教师可访问
- 前端路由守卫根据角色重定向

### 3.2 AI 图像生成

请求流程：
1. 学生在前端选择模型、输入提示词
2. 前端发送请求到后端 `/api/generate`
3. 后端通过 `GatewayAiClient` 调用网关的 `/v1/images/generations`
4. 网关根据模型 ID 路由到对应上游服务商
5. 生成结果返回并持久化到数据库

支持的生成模式：
- **文生图 (Text-to-Image)**：纯文本提示词生成图像
- **图生图 (Image-to-Image)**：基于参考图 + 文本提示词生成新图像

### 3.3 AI 学伴 (Tutor)

- 教师可配置学伴使用的文本模型和系统提示词
- 学生通过对话界面与 AI 导师交互
- 调用网关的 `/v1/chat/completions` 接口
- 支持多轮对话上下文

### 3.4 模型管理与同步

- 后端定义 `Model` 实体，存储模型元数据
- 教师可从网关同步模型列表（调用 `/model/info` 或 `/v1/models`）
- 同步后教师可配置模型的展示名称、描述、可用尺寸、是否开放

### 3.5 作业系统

- 教师创建作业（标题、描述、参考图、截止日期）
- 学生查看作业 → 创作 → 提交
- 教师查看提交 → 评分 → 反馈

## 四、数据模型

### 核心实体

| 实体 | 说明 |
|------|------|
| User | 用户（教师/学生），学生通过 teacherId 关联教师 |
| Model | AI 模型目录，存储模型 ID、类型、配置 |
| Generation | 生成记录，关联用户、模型、提示词、结果图 |
| Conversation | 对话记录（AI 学伴） |
| Assignment | 作业 |
| Submission | 学生作业提交 |
| Template | 提示词模板 |
| TutorConfig | AI 学伴配置（每教师一份） |
| GatewayConfig | 网关连接配置 |

## 五、API 接口概览

| 路径前缀 | 权限 | 说明 |
|---------|------|------|
| `/api/auth/**` | 公开 | 登录、注册 |
| `/api/public/**` | 公开 | 公开资源 |
| `/api/teacher/**` | 教师 | 教师管理功能 |
| `/api/generate` | 已认证 | 图像生成 |
| `/api/gallery` | 已认证 | 画廊查询 |
| `/api/conversations` | 已认证 | AI 学伴对话 |

## 六、设计语言

平台采用"Warm Editorial"设计语言，灵感来源于 Anthropic Claude 的视觉风格：
- 奶油色画布底色 (#faf9f5)
- 珊瑚色主色调 (#cc785c)
- Copernicus 衬线展示字体
- 深海军蓝产品表面
- 温暖人文主义的视觉调性
