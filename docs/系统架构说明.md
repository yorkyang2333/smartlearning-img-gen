# 晋彩智绘 — 系统架构与模型说明

## 一、系统架构

### 1.1 整体架构

```
┌─────────────────┐     JWT      ┌──────────────────┐  标准兼容接口  ┌─────────────────┐
│   Vue 3 前端     │ ──────────▶ │  Spring Boot 后端  │ ────────────▶ │  AI Gateway     │
│   (Port 5173)   │ ◀────────── │   (Port 8080)     │ ◀──────────── │  (Port 4000)    │
└─────────────────┘    JSON      └──────────────────┘   JSON         └─────────────────┘
                                         │                                    │
                                         ▼                                    ▼
                                  ┌──────────────┐                   ┌──────────────────┐
                                  │   MySQL 8    │                   │  上游 AI 服务商    │
                                  │(jincai_zhihui)│                   │ DeepSeek 等国产   │
                                  └──────────────┘                   │  大模型服务       │
                                                                     └──────────────────┘
```

### 1.2 三层解耦设计

1. **前端层**：Vue 3 + TypeScript + Pinia 状态管理 + Vue Router 路由守卫，负责 UI 渲染和角色隔离
2. **业务层**：Spring Boot 3.2.5 + Spring Security (JWT) + Spring Data JPA，负责认证鉴权、业务逻辑、数据持久化
3. **AI 路由层**：New API Gateway (Docker 单容器，内嵌 SQLite)，统一管理多家 AI 服务商的 API Key、渠道路由和重试机制

后端不直接调用任何 AI 服务商 API，所有 AI 请求统一通过 Gateway 的标准兼容接口转发。

## 三、核心模块说明

### 3.1 认证与鉴权

- 基于 JWT 的无状态认证，Token 有效期 24 小时
- 角色分为 `TEACHER` 和 `STUDENT`
- 后端路由权限（SecurityConfig）：
  - `/api/auth/**` — 公开（登录）
  - `/api/public/**` — 公开
  - `/api/teacher/**` — 仅 TEACHER 角色
  - 其余所有接口 — 需认证
- 前端路由守卫根据 `meta.role` 重定向，教师无法访问 `/student/*`，学生无法访问 `/teacher/*`

### 3.2 AI 图像生成

请求流程：
1. 学生在前端选择模型、输入提示词（可选上传参考图）
2. 前端发送请求到后端 `/api/generate/text-to-image` 或 `/api/generate/image-to-image`
3. 后端通过 `GatewayAiClient` 调用网关的 `/v1/images/generations`
4. 网关根据模型 ID 路由到对应上游服务商
5. 生成结果返回并持久化到 `Generation` 表

支持的生成模式：
- **文生图 (Text-to-Image)**：纯文本提示词生成图像
- **图生图 (Image-to-Image)**：基于参考图 + 文本提示词生成新图像

### 3.3 AI 学伴 (Tutor)

- 教师通过 Settings 页面配置学伴使用的模型和系统提示词（存储在 `TutorConfig` 表）
- 学生通过 TutorDrawer 组件与 AI 导师对话
- 支持两种调用方式：
  - `/api/generate/tutor-chat` — 普通请求-响应
  - `/api/generate/tutor-chat-stream` — SSE 流式输出
- 调用网关的 `/v1/chat/completions` 接口，支持多轮对话上下文

### 3.4 提示词工具

- **提示词构建器 (PromptBuilder)**：结构化表单辅助编写提示词
- **提示词优化 (PromptOptimizer / PromptOptimizerPopover)**：调用文本模型对提示词进行改写扩写
- **提示词分析 (analyze-prompt)**：对提示词进行质量分析
- 后端接口：`/api/generate/optimize-prompt`、`/api/generate/analyze-prompt`

### 3.5 模型管理与同步

- 后端 `Model` 实体存储模型元数据（ID、类型、服务商、配置 JSON）
- 教师点击"同步 AI Gateway 模型"触发 `GatewayModelSyncService`
- 同步优先调用 `/model/info`，失败回退到 `/v1/models`
- 同步后教师可配置：展示名称、描述、可用尺寸、是否开放

### 3.6 作业系统

- 教师创建作业（标题、描述、参考图、截止日期）
- 学生查看作业列表 → 进入 AssignmentPlay 创作 → 提交
- 教师查看提交 → 评分 → 文字反馈
- 相关实体：`Assignment`、`Submission`

### 3.7 AI 作品评审 (TutorReview)

- 对学生生成的图像进行多维度评审
- 调用多模态模型分析图像的光影构图、艺术风格、内容完整性等
- 后端接口：`/api/generate/review`

## 四、数据模型

### 核心实体

| 实体 | 说明 |
|------|------|
| User | 用户（TEACHER / STUDENT），学生通过 `teacherId` 关联教师 |
| Model | AI 模型目录，存储模型 ID、类型 (TEXT_TO_IMAGE / BOTH / TEXT_GENERATION)、服务商、配置 JSON |
| Generation | 生成记录，关联用户、模型、提示词、结果图、API 响应 |
| Conversation | 对话记录（AI 学伴），关联用户 |
| Assignment | 作业（教师创建） |
| Submission | 学生作业提交 |
| Template | 提示词模板（教师创建） |
| TutorConfig | AI 学伴配置（每教师一份），含模型名和系统提示词 |
| GatewayConfig | 网关连接配置（地址 + API Key） |
| ApiEndpoint | API 端点配置 |
