# 晋彩智绘 — 用户手册

## 一、产品简介

晋彩智绘是一款专为美术教学场景设计的 AI 图像生成平台。平台采用教师-学生双角色架构：教师管理 AI 模型、布置创作作业、监控学生活动并批改作品；学生通过对话式界面进行 AI 图像创作，并可获得 AI 导师的即时辅导。

**核心功能：**

- **角色分离**：教师端（模型管理、学生管理、作业批改、实时监控）与学生端（AI 创作、AI 导师、班级画廊、作业提交）完全独立
- **AI 图像生成**：支持文生图 (Text-to-Image) 和图生图 (Image-to-Image) 两种工作流
- **AI 导师**：基于大语言模型的对话式辅导，支持 SSE 流式输出，即时为学生提供创作反馈
- **AI 作品评审**：多模态模型从光影构图、艺术风格、内容完整性等维度评审学生作品
- **提示词工具**：提示词构建器 + AI 提示词优化，辅助学生表达创意
- **统一网关路由**：通过 New API Gateway 统一管理多家国产 AI 服务商
- **作业系统**：教师创建作业 → 学生创作提交 → 教师评分反馈，完整闭环

## 二、环境部署

### 2.1 环境依赖

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Java JDK | 17+ | 推荐 Eclipse Temurin 17 |
| Maven | 3.8+ | 后端构建工具 |
| Node.js | 20+ | 前端运行环境（含 npm） |
| MySQL | 8.0+ | 业务数据库 |
| Docker | 20+ | 运行 AI Gateway 容器 |

### 2.2 安装依赖（macOS）

```bash
# 安装 Homebrew（如未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Java 17
brew install openjdk@17

# 安装 Maven
brew install maven

# 安装 Node.js 20
brew install node@20

# 安装 MySQL 8
brew install mysql
brew services start mysql

# 安装 Docker Desktop
# 前往 https://www.docker.com/products/docker-desktop/ 下载安装
```

### 2.3 安装依赖（Linux / Ubuntu）

```bash
# 安装 Java 17
sudo apt update
sudo apt install openjdk-17-jdk

# 安装 Maven
sudo apt install maven

# 安装 Node.js 20（通过 NodeSource）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs

# 安装 MySQL 8
sudo apt install mysql-server
sudo systemctl start mysql

# 安装 Docker
sudo apt install docker.io docker-compose-plugin
sudo systemctl start docker
sudo usermod -aG docker $USER  # 免 sudo 运行 docker
```

### 2.4 数据库初始化

```bash
mysql -u root -e "CREATE DATABASE IF NOT EXISTS jincai_zhihui CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

数据库配置要求：MySQL 运行在 `localhost:3306`，使用 `root` 用户、空密码。如需修改，请编辑 `backend/src/main/resources/application.yml` 中的 `spring.datasource` 配置。

首次启动后端时，Hibernate 自动建表（`ddl-auto: update`），`DataSeeder` 自动写入默认账号和模型目录。

### 2.5 一键启动

```bash
sh start.sh
```

脚本按顺序启动三个服务：

1. **AI Gateway** (Docker 容器，端口 4000) — 统一 AI 模型路由网关
2. **Spring Boot 后端** (端口 8080) — 业务 API 服务
3. **Vue 前端** (Vite 开发服务器，端口 5173) — 用户界面

首次启动时前端会自动执行 `npm install` 安装依赖。按 `Ctrl+C` 可安全关闭所有服务。

### 2.6 单独启动各组件

```bash
# AI Gateway（需要 Docker 运行中）
./scripts/gateway/start_gateway.sh    # 启动
./scripts/gateway/stop_gateway.sh     # 停止

# 后端
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev -DskipTests

# 前端
cd frontend
npm install    # 首次运行需要
npm run dev
```

### 2.7 访问入口

| 服务 | 地址 | 账号 / 密码 |
|------|------|-------------|
| 应用前端 | http://localhost:5173 | teacher / 123456 或 student / 123456 |
| 后端 API | http://localhost:8080 | — |
| AI 网关管理面板 | http://localhost:4000 | root / 12345678 |

测试账号由 `DataSeeder` 在首次启动（users 表为空时）自动创建：
- 教师账号：用户名 `teacher`，密码 `123456`，显示名"王老师"
- 学生账号：用户名 `student`，密码 `123456`，显示名"小明"，自动关联到上述教师

### 2.8 生产构建

```bash
# 前端构建
cd frontend
npm run build    # 产物输出到 frontend/dist/

# 后端打包
cd backend
mvn package -DskipTests    # 产物输出到 backend/target/*.jar

# 运行 jar
java -jar backend/target/backend-0.0.1-SNAPSHOT.jar
```

## 三、教师端使用指南

### 3.1 登录

访问 http://localhost:5173，输入教师账号密码登录。登录成功后自动跳转到教师仪表盘 `/teacher/dashboard`。

### 3.2 仪表盘 (Dashboard)

教师首页，展示：

- 学生总数与活跃情况
- 近期生成统计（每日/每周）
- 作业完成进度概览

### 3.3 模型目录 (Models)

管理平台可用的 AI 模型：

1. **同步网关模型**：点击"同步 AI Gateway 模型"按钮，自动从网关拉取可用模型列表到本地
2. **模型配置**：为每个模型设置展示名称、教学描述、支持的生成尺寸
3. **模型启停**：控制哪些模型对学生可见（开关切换）
4. **批量操作**：支持批量启用/禁用模型
5. **网关管理**：点击"AI Gateway 网关管理"配置网关连接地址和 API Key
6. **渠道查看**：查看网关中已配置的上游渠道状态

### 3.4 学生管理 (Students)

- 查看所有学生列表（含活跃状态、生成次数）
- 添加单个学生账号（设置用户名、密码、显示名）
- 批量添加学生（一次创建多个账号）
- 启用/禁用学生账号
- 设置学生生成配额（限制每日生成次数）
- 重置学生密码
- 删除学生

### 3.5 作业管理 (Assignments)

- **创建作业**：设置标题、描述、参考图片（可选）、截止日期
- **查看提交**：浏览学生提交的作品列表
- **批改评分**：为学生作品打分（百分制）并给出文字反馈

### 3.6 模板管理 (Templates)

创建和管理提示词模板，供学生在创作时参考使用。教师可预设优质提示词模板，帮助学生学习如何编写有效的提示词。

### 3.7 实时监控 (Live)

实时查看学生当前的生成活动流，了解课堂进度。可看到哪些学生正在创作、使用了什么提示词、生成了什么图像。

### 3.8 设置 (Settings)

配置 AI 导师：

- **选择对话模型**：从已同步的文本模型中选择 AI 导师使用的模型（默认 `deepseek-v4-flash`）
- **编辑系统提示词**：自定义 AI 导师的角色设定和辅导风格，让导师的回复贴合教学目标

## 四、学生端使用指南

### 4.1 登录

访问 http://localhost:5173，输入学生账号密码登录。登录成功后自动跳转到创作工作区 `/student/generate`。

### 4.2 创作工作区 (Workspace)

核心创作界面，采用对话式交互：

1. **选择模型**：从教师开放的模型中选择（下拉菜单显示模型名称和描述）
2. **输入提示词**：在输入框中描述想要生成的图像内容
3. **提示词工具**：
   - **提示词构建器 (PromptBuilder)**：通过结构化表单（主题、风格、色调、构图等）辅助编写提示词
   - **提示词优化 (PromptOptimizer)**：点击优化按钮，AI 自动改写扩写提示词，提升生成质量
   - **提示词助手 (PromptHelper)**：提供编写建议和灵感
4. **生成图像**：点击发送按钮，等待 AI 生成结果（通常数秒到十几秒）
5. **图生图**：上传参考图片，基于参考图 + 文本提示词进行二次创作
6. **对话管理**：左侧面板可创建多个对话、切换历史对话、删除不需要的对话

### 4.3 AI 导师 (Tutor)

点击界面右侧的 AI 学伴按钮，打开 TutorDrawer 对话抽屉：

- 向 AI 导师提问创作技巧（如"水彩画怎么表现光影？"）
- 获取提示词编写建议（如"帮我优化这个提示词"）
- 了解艺术风格和构图知识（如"什么是黄金分割构图？"）
- 支持 SSE 流式输出，回复内容实时逐字显示
- 支持多轮对话，AI 导师记住上下文

### 4.4 AI 作品评审 (TutorReview)

生成图像后可请求 AI 评审：

- 点击作品下方的"AI 评审"按钮
- AI 从光影构图、艺术风格、内容完整性等维度评分
- 提供具体的改进建议
- 帮助学生在教师正式批改前了解作品优缺点

### 4.5 班级画廊 (Class Gallery)

浏览本班同学的生成作品：

- 瀑布流展示所有同学的优秀作品
- 获取创作灵感
- 支持点赞互动

### 4.6 我的作业 (Assignments)

- 查看教师布置的作业列表（含截止日期、完成状态）
- 点击进入作业详情查看具体要求和参考图
- 进入 AssignmentPlay 按要求进行创作
- 选择满意的作品提交，等待教师评分

## 五、AI Gateway 管理

### 5.1 访问网关面板

访问 http://localhost:4000，使用管理员账号登录：

- 用户名：`root`
- 密码：`12345678`

### 5.2 添加上游渠道

1. 进入"渠道"页面
2. 点击"添加新渠道"
3. 选择渠道类型（支持 DeepSeek、智谱、通义千问等国产大模型服务商）
4. 填入对应服务商的 API Key
5. 选择该渠道支持的模型
6. 保存并测试连接

### 5.3 默认模型配置

`DataSeeder` 首次启动时自动写入以下默认模型：

| 模型 ID | 类型 | 用途 |
|---------|------|------|
| `deepseek-v4-flash` | TEXT_GENERATION | AI 导师对话（默认） |
| 文生图模型 | TEXT_TO_IMAGE / BOTH | 文生图 / 图生图 |
| 多模态文本模型 | TEXT_GENERATION | 多模态分析 / AI 导师 |

实际模型 ID 以网关中配置的渠道为准，教师可通过"同步 AI Gateway 模型"拉取最新列表。

### 5.4 网关配置优先级

后端解析网关连接信息的顺序：

1. **数据库配置**：教师在"模型目录 → AI Gateway 网关管理"中保存的地址和 Key
2. **环境变量**：`NEWAPI_BASE_URL` / `NEWAPI_API_KEY`
3. **默认值**：`http://localhost:4000` / 内置开发 Key

即：数据库配置 > 环境变量 > 默认值。一般情况下使用默认值即可。

## 六、常见问题

| 问题 | 解决方案 |
|------|---------|
| 后端启动报数据库连接失败 | 确认 MySQL 已启动且存在 `jincai_zhihui` 数据库。默认配置为 root 用户空密码，如有不同请修改 `application.yml` |
| 网关容器启动失败 | 确认 Docker 已运行（执行 `docker ps` 检查）。如端口 4000 被占用，可修改 `docker-compose.gateway.yml` 中的端口映射 |
| 前端 API 请求 CORS 报错 | 确认后端运行在 8080 端口、前端在 5173 端口。后端 CORS 仅允许 `localhost:5173` 和 `localhost:3000` |
| 模型同步为空 | 先在网关面板（http://localhost:4000）中添加至少一个上游渠道并配置模型 |
| AI 导师无响应 | 确认教师设置中配置的导师模型在网关中可用（渠道已添加且状态正常） |
| `mvn` 命令未找到 | macOS 执行 `brew install maven`；Linux 执行 `sudo apt install maven` |
| `npm` 命令未找到 | 安装 Node.js 20+（macOS: `brew install node@20`；Linux: 通过 NodeSource 安装） |
| 前端启动后页面空白 | 检查浏览器控制台错误。常见原因：后端未启动导致 API 请求失败 |
| 学生登录后看不到模型 | 教师需在"模型目录"中将模型设为启用状态 |
| 生成图像超时 | 检查网关渠道状态是否正常，上游服务商 API Key 是否有效且有余额 |
