# 晋彩智绘 AI 图像生成平台 — 创作说明

## 一、创作背景

本项目源于与学校美术老师的深入交流。在与老师商讨业务流程和用户故事的过程中，了解到当前美术教学面临的痛点：学生创作后缺乏即时、个性化的反馈指导，教师精力有限难以逐一辅导每位学生。基于这一真实教学需求，本人设计并开发了"晋彩智绘"平台，将 AI 导师引入美术创作教学场景，让每位学生都能获得即时的多维度评审与对话式辅导。

## 二、原创与参考情况

本作品为本人独立原创开发，非团队作品。

- **前端**：Vue 3 + TypeScript + Vite 8 + Tailwind CSS 3.4 + Pinia 3 构建
- **后端**：Spring Boot 3.2.5 + Java 17 + Spring Data JPA + Spring Security (JWT)
- **AI 路由**：通过自部署的 New API 统一网关（Docker 容器）对接上游国产大模型服务

产品设计、交互流程、UI 实现、前后端业务代码及网关部署均由本人独立完成。

## 三、AI 工具使用情况

以下为平台中嵌入的 AI 功能，均作为产品特性服务于教学场景：

### 1. AI 学伴/导师（核心功能）

经统一网关调用国产大语言模型（默认 deepseek-v4-flash），对学生创作进行对话式辅导。支持 SSE 流式输出，呈现于学生工作台的 TutorDrawer 抽屉面板中。该功能直接回应了美术老师"希望学生能获得即时反馈"的核心诉求。教师可在设置页面自定义 AI 导师使用的模型和系统提示词，灵活调整辅导风格和知识范围。

### 2. AI 作品评审 (TutorReview)

调用多模态模型对学生生成的图像从光影构图、艺术风格、内容完整性等维度进行评审打分，提供改进建议。让学生在教师批改之前就能获得即时的专业反馈。

### 3. 图像生成

经统一网关调用国产图像生成模型，将学生提示词转化为图像作品，支持文生图和图生图两种模式。

### 4. 提示词优化

调用大语言模型对学生输入的提示词进行结构化改写与扩写，辅助学生更准确地表达创意，呈现于 PromptOptimizer 组件中。

> 所有 AI 模型仅作为辅助创作与教学的工具嵌入产品功能中，不参与项目本身的代码、设计与文档创作。

## 四、产品简介

晋彩智绘是一款专为美术教学场景设计的 AI 图像生成平台。平台采用教师-学生双角色架构：教师管理 AI 模型、布置创作作业、监控学生活动并批改作品；学生通过对话式界面进行 AI 图像创作，并可获得 AI 导师的即时辅导。

**核心功能：**

- **角色分离**：教师端（模型管理、学生管理、作业批改、实时监控）与学生端（AI 创作、AI 导师、班级画廊、作业提交）完全独立
- **AI 图像生成**：支持文生图 (Text-to-Image) 和图生图 (Image-to-Image) 两种工作流
- **AI 导师**：基于多模态大语言模型的对话式辅导，支持 SSE 流式输出
- **提示词工具**：提示词构建器 + AI 提示词优化，辅助学生表达创意
- **统一网关路由**：通过 New API Gateway 统一管理多家国产 AI 服务商，后端不直接对接任何上游 API
- **作业系统**：教师创建作业 → 学生创作提交 → 教师评分反馈，完整闭环

## 七、核心模块详细说明

### 7.1 AI 导师系统（核心亮点）

AI 导师是本平台的核心功能，直接解决美术教学中"学生缺乏即时反馈"的痛点。

**工作原理：**

- 教师通过 Settings 页面配置 AI 导师使用的模型和系统提示词（存储在 `TutorConfig` 表）
- 学生通过 TutorDrawer 组件与 AI 导师对话
- 支持两种调用方式：普通请求-响应模式和 SSE 流式输出模式
- 调用网关的 `/v1/chat/completions` 接口，支持多轮对话上下文

**教学价值：**

- **即时性**：学生创作后可立即获得反馈，无需等待教师逐一批改
- **个性化**：AI 导师根据学生具体作品和提问给出针对性指导
- **多维度**：从创作技巧、提示词编写、艺术风格、构图知识等多角度辅导
- **可定制**：教师可自定义系统提示词，让 AI 导师的辅导风格和知识范围贴合教学目标

**学生使用方式：**

- 点击界面的 AI 学伴按钮，打开 TutorDrawer 对话抽屉
- 向 AI 导师提问创作技巧
- 获取提示词编写建议
- 了解艺术风格和构图知识
- 支持 SSE 流式输出，实时显示回复

### 7.2 AI 作品评审 (TutorReview)

对学生生成的图像进行多维度评审：

- 调用多模态模型分析图像的光影构图、艺术风格、内容完整性等
- 从多个维度给出评分和改进建议
- 让学生在教师正式批改前获得即时专业反馈
- 后端接口：`/api/generate/review`

### 7.3 AI 图像生成

**请求流程：**

1. 学生在前端选择模型、输入提示词（可选上传参考图）
2. 前端发送请求到后端 `/api/generate/text-to-image` 或 `/api/generate/image-to-image`
3. 后端通过 `GatewayAiClient` 调用网关的 `/v1/images/generations`
4. 网关根据模型 ID 路由到对应上游服务商
5. 生成结果返回并持久化到 `Generation` 表

**支持的生成模式：**

- **文生图 (Text-to-Image)**：纯文本提示词生成图像
- **图生图 (Image-to-Image)**：基于参考图 + 文本提示词生成新图像

### 7.4 提示词工具

- **提示词构建器 (PromptBuilder)**：结构化表单辅助编写提示词
- **提示词优化 (PromptOptimizer / PromptOptimizerPopover)**：调用文本模型对提示词进行改写扩写
- **提示词分析 (analyze-prompt)**：对提示词进行质量分析
- 后端接口：`/api/generate/optimize-prompt`、`/api/generate/analyze-prompt`

### 7.5 认证与鉴权

- 基于 JWT 的无状态认证，Token 有效期 24 小时
- 角色分为 `TEACHER` 和 `STUDENT`
- 后端路由权限（SecurityConfig）：
  - `/api/auth/**` — 公开（登录）
  - `/api/public/**` — 公开
  - `/api/teacher/**` — 仅 TEACHER 角色
  - 其余所有接口 — 需认证
- 前端路由守卫根据角色重定向，教师无法访问学生页面，学生无法访问教师页面

### 7.6 作业系统

- 教师创建作业（标题、描述、参考图、截止日期）
- 学生查看作业列表 → 进入 AssignmentPlay 创作 → 提交
- 教师查看提交 → 评分 → 文字反馈
- 相关实体：`Assignment`、`Submission`

### 7.7 模型管理与同步

- 后端 `Model` 实体存储模型元数据（ID、类型、服务商、配置 JSON）
- 教师点击"同步 AI Gateway 模型"触发 `GatewayModelSyncService`
- 同步优先调用 `/model/info`，失败回退到 `/v1/models`
- 同步后教师可配置：展示名称、描述、可用尺寸、是否开放

## 九、AI Gateway 接入说明

### 9.1 概述

晋彩智绘通过 New API Gateway（Docker 单容器）统一管理所有上游国产 AI 服务商。后端不直接调用任何服务商 API，所有请求经网关的标准兼容接口转发。

### 9.2 配置优先级

网关连接信息的解析顺序：

1. **数据库 `GatewayConfig` 表**（教师在"模型目录 → AI Gateway 网关管理"中保存的配置）
2. **环境变量** `NEWAPI_BASE_URL` / `NEWAPI_API_KEY`
3. **application.yml 默认值**（`http://localhost:4000` / 内置开发 Key）

即：数据库配置 > 环境变量 > 默认值。

### 9.3 后端调用接口

| 用途 | 接口 |
|------|------|
| 聊天 / 多模态分析 / AI 导师 | `POST {GATEWAY}/v1/chat/completions` |
| 文生图 / 图生图 | `POST {GATEWAY}/v1/images/generations` |
| 模型同步（优先） | `GET {GATEWAY}/model/info` |
| 模型同步（回退） | `GET {GATEWAY}/v1/models` |

所有请求通过 `GatewayAiClient` 或 `NewApiGatewayClient` 发出，网关地址由 `GatewayConfigService` 按上述优先级解析。

### 9.4 部署方式

仓库自带 Docker 自动化脚本：

```bash
# 单独启停网关
./scripts/gateway/start_gateway.sh
./scripts/gateway/stop_gateway.sh

# 一键启动整个项目（含网关）
sh start.sh
```

启动后在根目录生成 `.gateway/data/` 工作区，存放 New API 的 SQLite 数据库和配置。

### 9.5 网关渠道配置建议

在 New API 管理面板中添加上游渠道（支持 DeepSeek、智谱、通义千问等国产大模型服务商），推荐配置以下模型：

| 模型 ID | 用途 |
|---------|------|
| `deepseek-v4-flash` | AI 导师对话（高速） |
| 文生图模型 | 文生图 / 图生图 |
| 多模态文本模型 | AI 导师对话 / 多模态分析 |

以上为示例，实际模型 ID 以网关中配置的渠道和模型别名为准。

## 十一、用户手册

### 11.1 教师端使用指南

**登录**：访问 http://localhost:5173，使用教师账号登录（默认：teacher / 123456）。登录后自动跳转到教师仪表盘。

**仪表盘 (Dashboard)**：教师首页，展示学生总数与活跃情况、近期生成统计（每日/每周）、作业完成进度概览。

**模型目录 (Models)**：

- 同步网关模型：点击"同步 AI Gateway 模型"，自动从网关拉取可用模型列表
- 模型配置：为每个模型设置展示名称、描述、支持的生成尺寸
- 模型启停：控制哪些模型对学生可见
- 批量操作：支持批量启用/禁用模型
- 网关管理：配置 AI Gateway 的连接地址和 API Key
- 渠道查看：查看网关中已配置的上游渠道状态

**学生管理 (Students)**：查看所有学生列表、添加单个学生账号、批量添加学生、启用/禁用学生账号、设置学生生成配额、重置学生密码、删除学生。

**作业管理 (Assignments)**：创建作业（设置标题、描述、参考图片、截止日期）、查看提交（浏览学生提交的作品）、批改评分（为学生作品打分并给出文字反馈）。

**模板管理 (Templates)**：创建和管理提示词模板，供学生在创作时参考使用。

**实时监控 (Live)**：实时查看学生当前的生成活动流，了解课堂进度。

**设置 (Settings)**：配置 AI 导师——选择对话模型、编辑系统提示词。默认导师模型为 `deepseek-v4-flash`，可切换为任意已同步的文本模型。

### 11.2 学生端使用指南

**登录**：访问 http://localhost:5173，使用学生账号登录（默认：student / 123456）。登录后自动跳转到创作工作区。

**创作工作区 (Workspace)**：核心创作界面，采用对话式交互。

- 选择模型：从教师开放的模型中选择（支持文生图和图生图模型）
- 输入提示词：描述想要生成的图像内容
- 提示词工具：提示词构建器辅助编写、AI 提示词优化、提示词助手提供建议
- 生成图像：点击发送，等待 AI 生成结果
- 图生图：上传参考图片，基于参考进行二次创作
- 对话管理：可创建多个对话，切换和删除历史对话

**AI 导师 (Tutor)**：点击界面的 AI 学伴按钮，打开 TutorDrawer 对话抽屉，向 AI 导师提问创作技巧、获取提示词编写建议、了解艺术风格和构图知识。支持 SSE 流式输出，实时显示回复。

**AI 作品评审 (TutorReview)**：生成图像后可请求 AI 评审，从光影构图、艺术风格、内容完整性等维度评分，并提供改进建议。

**班级画廊 (Class Gallery)**：浏览本班同学的生成作品，获取创作灵感，支持点赞互动。

**我的作业 (Assignments)**：查看教师布置的作业列表、进入作业详情查看要求、进入 AssignmentPlay 按要求创作、提交作品等待教师评分。

## 十五、常见问题

| 问题 | 解决方案 |
|------|---------|
| 后端启动报数据库连接失败 | 确认 MySQL 已启动，且存在 `jincai_zhihui` 数据库（root 用户空密码） |
| 网关容器启动失败 | 确认 Docker 已运行（`docker ps`） |
| 前端 API 请求 CORS 报错 | 确认后端运行在 8080 端口，前端在 5173 端口（CORS 仅允许这两个源） |
| 模型同步为空 | 先在网关面板中添加至少一个上游渠道并配置模型 |
| AI 导师无响应 | 确认 TutorConfig 中配置的模型在网关中可用 |
