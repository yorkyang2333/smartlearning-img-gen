# 创作说明

## 创作背景

本项目源于与学校美术老师的深入交流。在与老师商讨业务流程和用户故事的过程中，了解到当前美术教学面临的痛点：学生创作后缺乏即时、个性化的反馈指导，教师精力有限难以逐一辅导每位学生。基于这一真实教学需求，本人设计并开发了"晋彩智绘"平台，将 AI 导师引入美术创作教学场景，让每位学生都能获得即时的多维度评审与对话式辅导。

## 原创与参考情况

本作品为本人独立原创开发，非团队作品。

- **前端**：Vue 3 + TypeScript + Vite 8 + Tailwind CSS 3.4 + Pinia 3 构建
- **后端**：Spring Boot 3.2.5 + Java 17 + Spring Data JPA + Spring Security (JWT)
- **AI 路由**：通过自部署的 New API 统一网关（Docker 容器）对接上游大模型服务

产品设计、交互流程、UI 实现、前后端业务代码及网关部署均由本人独立完成。

## AI 工具使用情况

以下为平台中嵌入的 AI 功能，均作为产品特性服务于教学场景：

1. **AI 学伴（核心功能）**：经统一网关调用大语言模型（默认 deepseek-v4-flash），对学生创作进行对话式辅导。支持 SSE 流式输出，呈现于学生工作台的 TutorDrawer 抽屉面板中。该功能直接回应了美术老师"希望学生能获得即时反馈"的核心诉求。

2. **AI 作品评审 (TutorReview)**：调用多模态模型对学生生成的图像从光影构图、艺术风格、内容完整性等维度进行评审打分，提供改进建议。

3. **图像生成**：经统一网关调用图像生成模型（如 gpt-image-2、dall-e-3、gemini/gemini-3.1-flash-image-preview），将学生提示词转化为图像作品，支持文生图和图生图两种模式。

4. **提示词优化**：调用大语言模型对学生输入的提示词进行结构化改写与扩写，辅助学生更准确地表达创意，呈现于 PromptOptimizer 组件中。

所有 AI 模型仅作为辅助创作与教学的工具嵌入产品功能中，不参与项目本身的代码、设计与文档创作。
